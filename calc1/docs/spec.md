# ゲーム仕様: Calc Rush (仮)

## ゲーム名（仮称可）
- Calc Rush（仮）

## コンセプト（1-3 行）
- 1プレイぶんの演算選択列を最初からすべて見渡し、現在値を目的の数字へ一致させる算数ゲーム。
- すべての手は初期状態でスキップになっており、プレイヤーは任意の手を自由な順序で編集できる。
- 全体制限時間内に試行錯誤しながら解を作る、短時間プレイ向けのミニゲームとする。

## ターゲットプレイヤー
- 小学生高学年から大人までの、暗算やパズルが好きなプレイヤー。
- 1プレイを短時間で終えたいカジュアルプレイヤー。

## コアループ（プレイ中に繰り返す行動）
1. 目的の数字（Target）と初期値（Start）を確認する。
2. レベルに応じた手数ぶんの演算選択列（各手に4候補 + スキップ）をすべて確認する。
3. すべてスキップで初期化された選択を、任意の手からクリック/タップで変更する。
4. 選択変更のたびに、先頭手から順に再計算した現在値（Current）と計算履歴を更新する。
5. 途中結果のいずれかで目標一致した時点で即時勝利とし、一致以降の手は自動でスキップ扱いにする。
6. 目標未達のまま制限時間（60秒）を超えた場合に敗北とする。

## 操作仕様（入力デバイスと主要操作）
- 対応入力: マウス（またはタッチ）のみ。
- マウス/タッチ: 任意の手の候補ボタンをクリック/タップして選択を変更。
- スキップ: 各手の「スキップ」候補をクリック/タップで選択。
- 初期状態: 全手がスキップ選択済み。
- 並び順自由編集: 第1手から順番に選ぶ必要はなく、どの手からでも変更可能。
- 即時勝利: 途中手で目標一致した時点でラウンドを終了し、以降の手は自動スキップとして確定する。
- 結果画面での確認: クリア/ゲームオーバー時に「確認」ボタンが表示される。ボタンをクリック/タップして結果を確認。
- リトライ: 確認後、クリア時は「次のレベルへ」、敗北時は「最初から」または「続きから」を選んでゲームを開始。

## 画面・Scene 構成
- MVP は 1 Scene 構成（Game Scene）とする。
- 表示要素:
  - 現在レベル（例: Lv.3）
  - 目的の数字
  - 初期値
  - 現在値
  - 残り時間
  - 演算選択列（全手ぶんを同時表示。各手は4候補 + スキップ）
  - 計算履歴（各手の行に1対1で対応し、同じ行内に結果を表示）
  - ステータスメッセージ（進行/勝利/敗北）
- レイアウト方針:
  - 上部中央: ステータス情報（レベル・目標値・初期値・現在値・残り時間）。
  - 中央: 行同期テーブル構成とし、各行で「手番号・演算選択肢・履歴結果」を横並び表示する。
  - 計算履歴は同じ手の選択肢と必ず同じ行に表示し、選択変更時に即時更新する。
  - スクロールは1つの共通コンテナで行い、選択列と履歴列の行ずれを発生させない。
  - 各手の行には「手番号」「選択中の演算」「候補ボタン」を明示し、現在選択を視覚的に強調する。
- レイアウト例（簡易）:

```text
┌──────────────────────────────────────────────────────────────────────┐
│ Lv.3  Target: 96  Start: 4  Current: 72                  Time: 00:41 │
├──────────────────────────────────────────────────────────────────────┤
│ 手01 | (○ Skip)  ＋2  ×3  ー1  ÷2 | 結果: 4   (Start=4, 変更なし)    │
│ 手02 |  Skip   (○ ×3) ＋1  ー2  ÷3 | 結果: 12  (4 ×3)               │
│ 手03 |  Skip    ＋5  ×2 (○ ー1) ÷2 | 結果: 11  (12 ー1)             │
│ 手04 |  Skip    ＋10 ー5  ×5 (○ ÷2) | 結果: 5   (11 ÷2 切り捨て)     │
│ ...  |                               | ... (同一行でスクロール)      │
│ 手12 |  Skip    ＋2  ー1  ×3  ÷3     | 結果: 72                      │
├──────────────────────────────────────────────────────────────────────┤
│ ステータスメッセージ: 目標まであと24。手03か手04の見直しがおすすめ。 │
└──────────────────────────────────────────────────────────────────────┘
```

## 難易度テーブル

| レベル | 初期値範囲 | 目標値範囲 | 最大手数 |
|---|---|---|---|
| 1 | 1〜5 | 20〜50 | 12 |
| 2 | 1〜10 | 50〜100 | 10 |
| 3 | 1〜15 | 80〜150 | 9 |
| 4 | 1〜20 | 100〜200 | 8 |
| 5以降 | 1〜20 | 150〜300 | 8（固定） |

- 制限時間: 1プレイ全体で60秒固定（全レベル共通）。
- クリア時にレベルを +1 し、localStorage に保存する。
- 敗北時は結果確認後に「最初から（Lv.1）」または「続きから（到達済みレベル）」を選択する。
- 「最初から」を選んだ場合は Lv.1 で開始し、localStorage のレベル値も 1 に更新する。
- 「続きから」を選んだ場合は到達済みレベルで再開する。
- ゲーム起動時は常に Lv.1 から開始する（localStorage からのレベル復元は行わない）。

## ルール（勝利条件、失敗条件、スコア条件）
- 勝利条件:
  - 先頭手から順に再計算した途中結果のいずれかで、現在値が目的の数字と一致する。
  - 一致した手より後ろの手は自動でスキップ扱いとなり、勝利時点の履歴を最終結果として確定する。
- 失敗条件:
  - 制限時間終了時点で現在値が目的の数字と一致していない。
- 目標値超過時:
  - 現在値が目標値を上回っても制限時間内であれば続行する。
  - 目標値ちょうどにならない限り勝利にならない。
  - 減算・除算で現在値を戻すことができる。
- タイムアウト時:
  - 残り時間が0になった時点で、各手の選択を確定して判定する。
  - 未変更の手はスキップのままとする（値は変化しない）。
  - 残り0秒と同時に入力された操作は有効とし、その入力を反映した後に勝敗判定を行う。
- スキップ操作:
  - 各手でプレイヤーは明示的にスキップを選択できる。
  - スキップはその手の演算未適用を意味し、再編集で他演算へ変更可能。
- スコア条件（MVP）:
  - スコアは未実装（勝敗のみ）。
- 問題の生成:
  - 初期値と目標値を決定した際に、操作プール内の演算のみを使用して、演算選択列上に必ず到達可能な選択パターンが存在するように生成する。
  - この仕様により、プレイヤーには「クリア可能性が常に存在する」ことを保証する。
  - ゲームオーバー時、その正解パターン（最短または推奨）を表示することで、プレイヤーに「実は解けていたかもしれない」を理解させ、再挑戦時の参考にできる。

## MVP 範囲（初版で作る範囲）

- 勝利/敗北判定。

- 問題生成時に必ず正解パターンが存在することを保証。

- 難易度テーブルに基づくレベル進行。
- レベルの localStorage 永続化。

## 演算プール

| 種別 | 演算 |
|---|---|
| 加算 | +1, +2, +5, +10 |
| 減算 | -1, -2, -5 |
| 乗算 | ×2, ×3, ×5 |
| 除算 | ÷2, ÷3（端数切り捨て） |

- 負の値になる演算は許可（現在値が0以下になっても続行）。
- 除算の端数は切り捨て（例: 7 ÷ 2 = 3）。
- 候補はこのプールからランダムに重複なく抽選する。
- 各手には5つの選択肢（演算4件 + スキップ1件）を持たせる。
- 同一手内の演算4件は重複禁止。
- 手と手の間で演算が重複することは許可する。
- ゲーム開始時に全手ぶんの候補を一括生成し、全手の初期選択はスキップに設定する。
- ボタンの表示文字列は全角記号を使用する（例: ＋1, ー2, ×5, ÷2）。

## 技術要件（Phaser/Vite/ESLint/Vitest での実装方針）
- Phaser 4 + TypeScript で実装する。
- ゲーム状態（初期値、現在値、目標値、手ごとの選択、全体タイマー状態、レベル）を Scene 内で明示管理する。
- 毎フレーム更新を最小化し、タイマーは Phaser Time Event を使用する。
- レベルの永続化は localStorage を使用する（キー名: `calcRushLevel`）。
- ビルドは Vite（静的出力）を前提とする。
- ESLint エラー 0 を維持する。
- Vitest でロジックの単体テストを実装する（可能な限り純粋関数化）。

## テスト観点（最低限の単体テスト対象）
- 演算適用ロジック:
  - 手ごとの選択列を先頭から順に適用した結果が期待どおりになる。
  - スキップ選択の手では現在値が変化しない。
  - 任意の手の選択を変更すると、再計算結果が即時反映される。
- 勝敗判定:
  - 途中結果で目標一致した時点で即時勝利。
  - 即時勝利後は後続手が自動スキップとして扱われる。
  - 制限時間終了時に不一致なら敗北。
  - 残り0秒と同時に入力された操作が判定に反映される。
- 難易度・レベル:
  - クリア時にレベルが +1 される。
  - 敗北時に「最初から」を選ぶと Lv.1 で再開する。
  - 敗北時に「続きから」を選ぶと到達済みレベルで再開する。
  - 各レベルで初期値・目標値・最大手数が難易度テーブルどおりに設定される。
  - レベルが localStorage に保存される。
- 候補生成:
  - 全手ぶんの候補が生成される。
  - 同一手内で候補重複が発生しない。
  - 不正な候補（未定義演算）が含まれない。
  - 初期状態で全手がスキップ選択になっている。
  - 手の編集順序に依存せず、同じ最終選択なら同じ結果になる。
