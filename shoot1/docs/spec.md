# shoot1 ゲーム仕様

## ゲーム名

**shoot1** ― 弾幕回避シューティング（仮称）

## コンセプト

画面下部の自機を操作し、上から降り注ぐ敵弾を避け続ける。
地雷を設置して敵機が地雷に当たると、敵機と敵弾を爆破で巻き込める。ただし自分が踏んでも爆発する。どれだけ長く生き残れるかを競う。

## ターゲットプレイヤー

- ブラウザゲームを手軽に遊びたいプレイヤー
- 弾幕系シューティングに親しみのあるプレイヤー

## コアループ

1. 自機をキーボードで上下左右に動かす
2. 敵弾が当たる前に避け続ける
3. 地雷を設置して、敵機が触れるか時間経過で爆発・周囲の弾を巻き込んで消去
4. 地雷に自分が触れても爆発してゲームオーバー（扱い注意）
5. 被弾またはミス踏みでゲームオーバー → スコア表示 → リスタート

## 操作仕様

### キーボード（PC）

| 操作 | 方法 |
|------|------|
| 移動 | 矢印キー（上下左右）または WASD |
| 地雷設置 | スペースキー／Z キー（自機の現在位置に設置） |
| リスタート | ゲームオーバー画面で R キーまたはクリック |

### 仮想コントローラ（スマートフォン・タッチデバイス）

| 操作 | 方法 |
|------|------|
| 移動 | 画面左下に表示される仮想スティック（8方向） |
| 地雷設置 | 画面右下に表示される仮想ボタン（BOMB） |
| リスタート | ゲームオーバー画面をタップ |

- 仮想コントローラはタッチデバイス検出時に自動表示する
- PC ではデフォルト非表示（キーボード操作優先）
- 仮想スティックはマルチタッチ対応（スティック操作中にボタンタップ可能）

| Scene | 役割 |

| `Boot` | アセット読み込み |
| `Game` | メインゲームプレイ |
| `GameOver` | スコア表示・リスタート |

## 敵機パターン

編隊型を中心に構成する。時間経過でパターンが切り替わり難易度が上がる。

| パターン名 | 動き | 弾種 | 出現タイミング |
|-----------|------|------|--------------|
| **横一列編隊** | 3〜5機が横並びで直下降下 | 直下弾 | 序盤（常時） |
| **V字編隊** | V字フォーメーションで降下 | 自機狙い弾 | 序盤〜中盤 |
| **くの字編隊** | 斜めに整列して斜め降下 | 直下弾 + N-way弾 | 中盤 |
| **波状編隊** | 横一列がsin波で揺れながら降下 | 全方位弾 | 中盤〜終盤 |
| **包囲編隊** | 画面両サイドから挟み込む形で進入 | 自機狙い弾 | 終盤 |

### 敵機の共通仕様

- 編隊は一定間隔（例: 5〜8秒）で画面上部から出現する
- 編隊内の各機は一定間隔で弾を撃ち、画面外に出ると消える
- 難易度は時間経過で段階的に上昇（弾速・弾数・編隊サイズを増加）
- 自機の弾による撃破は MVP 外（地雷での撃破のみ）

## 敵弾パターン

| パターン名 | 内容 | 登場タイミング |
|-----------|------|--------------|
| **直下弾** | 真下に一定速度で落下 | 序盤から |
| **自機狙い弾** | 発射時点の自機方向へ飛ぶ | 序盤から |
| **3-way弾** | 自機狙い方向を中心に3方向扇形 | 中盤から |
| **5-way弾** | 自機狙い方向を中心に5方向扇形 | 終盤から |
| **全方位弾** | 360度均等に8方向拡散 | 中盤から |
| **渦巻き弾** | 時間差で角度をずらしながら連射（渦巻き状） | 終盤から |
| **高速弾** | 2〜3倍速の直下弾（少量） | 中盤から |

### 弾パターンの難易度スケール

- **0〜30秒**: 直下弾・自機狙い弾のみ、低速
- **30〜60秒**: 3-way弾・全方位弾が混入、やや高速化
- **60〜120秒**: 5-way弾・渦巻き弾・高速弾が出現
- **120秒以降**: 全パターンが混在、弾速・密度が最大

## ルール

- **失敗条件**: 敵弾が自機に当たる、または自機が地雷に触れる（どちらも1回でゲームオーバー）
- **スコア条件**: 生存時間をスコア（秒×10）とする
- **地雷**:
  - ゲーム開始時に 3 発所持（残弾数を HUD 表示）
  - スペース／Z キーで自機の現在位置に設置
  - **当たり判定の対象は敵機のみ**（敵弾には反応しない）
  - 敵機が地雷に触れると爆発し、爆発半径内の全敵機・全敵弾を消去する
  - 自機が地雷に触れると即ゲームオーバー
  - **設置直後は自機との当たり判定を一定時間（0.5秒）無効化**し、置いた瞬間に自爆しないようにする
  - 敵機が来なくても、設置から一定時間（10秒）後に自動で爆発する（巻き込み効果あり）

## MVP 範囲（初版で作る範囲）

- 自機の移動（矢印キー・WASD）
- 敵機の編隊出現（横一列・V字の2パターン）
- 敵弾の生成（直下弾・自機狙い弾・3-way弾）
- 時間経過による難易度上昇（弾速・弾数増加）
- 敵弾との当たり判定（円形）
- 地雷設置（自機位置に配置、残弾3発）
- 地雷と敵機の当たり判定 → 爆発・周囲の敵機・敵弾を巻き込み消去
- 地雷と自機の当たり判定 → 即ゲームオーバー（設置直後0.5秒は無効）
- 地雷の時間経過爆発（10秒後に自動爆発、巻き込み効果あり）
- 生存スコア表示（HUD）
- 地雷残弾数の HUD 表示
- ゲームオーバー画面とリスタート
- 仮想コントローラ（タッチデバイス自動表示）
  - 仮想スティック（左下）
  - 地雷設置ボタン（右下）

## 非MVP範囲（今回は作らない範囲）

- 自機からの射撃（敵を倒す要素）
- BGM・効果音
- ハイスコアの永続保存
- 難易度設定
- 複数ライフ
- 渦巻き弾・包囲編隊・くの字編隊・波状編隊（非MVPパターン）
## 技術要件

- フレームワーク: Phaser 4.x（TypeScript）
- ビルド: Vite
- lint: ESLint（エラー0件）
- テスト: Vitest（カバレッジ90%以上）
- アセット: 図形描画（`Graphics`）で代替し、外部画像なしで動作させる
- ゲーム状態はシーン内のプロパティで管理し、グローバル汚染しない
- 物理エンジン: Phaser 4 の Arcade Physics（利用可能であることを確認済み）
- 仮想コントローラ: **VirtualJoystick プラグインは Phaser 4 非対応のため使用しない**
  - `Phaser.GameObjects.Ellipse` + `Phaser.Geom.Circle` + `Pointer` イベントで自前実装する（first-game の実装を参照）
  - 独立した UI Scene として重ねる
- レイアウト: `Scale.FIT` + `CENTER_BOTH` でスマートフォン縦横両対応

## ゲーム画面内の地雷ビジュアル仕様

- 地雷は黄色の円で表示（`Graphics`）
- 設置後7秒まで: 通常表示
- 残り3秒から: 点滅（点滅スピードが上がり緊迫感を表現）
- 同じ座標への重ね置きは禁止（地雷がすでにある場合は設置操作を無視）

## 爆発ビジュアル仕様

- 爆発半径を円で描画し、数フレームでフェードアウト（`Graphics` + `alpha`減算）
- 爆発時にカメラを軽くシェイク（`cameras.main.shake`）
- 爆発ビジュアルが完了する前に巻き込み対象の消去処理を先に実行する

## リスタートフロー

- GameOver → Game シーンを直接再起動（Boot を経由しない）
- `this.scene.start('Game')` でシーン内ステートを完全リセット
- アセット（Graphics のみ）は再読み込み不要

## 編隊挙動詳細

- 編隊内の一部が地雷で倒された場合、残りの敵機はそのまま降下し続ける（フォーメーションは崩れても継続）
- 編隊は機体開名度を持たず各機が独立して動く

- スコア計算ロジック（時間 × 係数）
- 地雷残弾数管理（初期値・設置で減少・0のとき設置不可）
- 自機の移動ロジック（境界クランプ）
- 敵弾生成ロジック（座標範囲、速度範囲）
- 当たり判定ロジック（円同士の重なり判定）
- 爆発の巻き込み判定（爆発半径内の敵機・敵弾を消去、範囲外は残る）
- 地雷タイムアウト爆発（10秒後に自動爆発）
- 自機が地雷に触れた場合のゲームオーバー判定
- 設置直後の自機判定無効化（0.5秒の猶予時間）
- 仮想スティック入力のベクトル変換（8方向 → 移動速度）
- タッチデバイス検出フラグの切り替え
- 難易度スケール（経過時間 → 弾速・弾数の段階変化）
- 編隊生成ロジック（横一列・V字の座標計算）
- N-way弾の角度計算（自機狙い方向を中心に扇形展開）
