# ゲーム名

editmin（仮）

# コンセプト（1-3 行）

マインスイーパーの盤面を自由に作成・検証するためのフィールドエディタ。
爆弾配置に応じて各マスの数字を即時再計算し、作問難易度を可視化する。
さらに、仮定なしの論理手順だけで解ける盤面かどうかを自動判定する。

# ターゲットプレイヤー

- マインスイーパーの問題を作成したいユーザー
- 論理的に解ける盤面を設計したいユーザー
- 難易度調整をしながら問題を公開したいユーザー

# コアループ（プレイ中に繰り返す行動）

1. 盤面サイズ（幅・高さ）を設定する
2. 最初に掘る想定マス（開始マス）を指定する
3. セルをクリックして爆弾を配置／解除する
4. 盤面の数字（隣接爆弾数）を確認する
5. 難易度と論理解法可否の判定結果を確認する
6. 必要に応じて配置を調整し、再判定する

# 操作仕様（入力デバイスと主要操作）

- 入力デバイス: マウス（必須）、キーボード（補助）
- 左クリック: 対象セルの爆弾配置をトグル
- 右クリック: 予備操作（将来拡張、初版では未使用または無効）
- UI入力: 幅・高さを数値入力して盤面サイズを変更
- 開始マス指定: 専用モードまたは UI ボタン選択後、セルクリックで開始マスを設定
- キーボード補助（任意）: フォーカス中セルに対して Space で爆弾トグル
- スクロール: 盤面が表示領域を超える場合、盤面表示エリア内の縦横スクロールで盤面全体を移動

開始マス制約:

- 開始マス自身とその周囲 8 マス（盤面内に存在するセル）には爆弾を置けない
- 既存爆弾が制約領域に含まれる場合、開始マス設定を拒否する（自動解除は行わない）

# 画面・Scene 構成

- EditorScene（単一 Scene）
  - 盤面表示エリア（スクロールコンテナ）
  - サイズ設定UI（幅・高さ）
  - 判定結果表示エリア（難易度、論理解法可否、補足メトリクス）
  - 補助UI（リセット、乱数生成は非MVP）

表示仕様:

- 画面サイズは可変（レスポンシブ）
- マスの描画サイズは固定（自動縮小しない）
- 盤面が表示領域に収まらない場合は、盤面表示エリア内スクロールで対応する（ページ全体スクロールには依存しない）

# ルール（勝利条件、失敗条件、スコア条件）

このプロジェクトは対戦/クリア型ゲームではなくエディタであるため、勝敗は定義しない。
代わりに以下を盤面品質指標として扱う。

- 数字整合性: 全セルの表示数字が爆弾配置と一致していること
- 開始マス安全性: 開始マスおよび周囲 8 マスに爆弾が存在しないこと
- 難易度ランク: 盤面の推定解法コストに応じてランク分け（例: Easy/Normal/Hard/Expert）
- 論理解法可否: 仮定（当て推量）なしで、基本ルールから高度ルールまで適用して全安全マスを特定可能か

難易度ランク定義（4段階固定）:

- Easy: 基本ルール中心で解法が進み、分岐候補が少ない
- Normal: 基本ルールに加えて複数の連鎖推論が必要
- Hard: 高度ルールの適用が複数回必要
- Expert: 高度ルールを前提にしても推論負荷が高く、長い連鎖を要する

難易度根拠スコア定義（初版）:

- 難易度スコア `D` は 0-100 の整数とする（小数は四捨五入）
- 盤面解析で以下の指標を計測する
  - `B`: 基本ルール適用ステップ数
  - `A`: 高度ルール適用ステップ数
  - `C`: 連鎖推論の最大深さ
  - `U`: 基本ルールのみ適用後に未確定で残る安全マス率（0.0-1.0）
- `U` の算出定義
  - `totalSafe = width * height - mineCount`
  - `unresolvedSafe = 基本ルールのみ適用後に未確定で残る安全マス数`
  - `U = unresolvedSafe / max(totalSafe, 1)`
- 正規化
  - `b = min(B / 120, 1.0)`
  - `a = min(A / 60, 1.0)`
  - `c = min(C / 20, 1.0)`
  - `u = min(max(U, 0.0), 1.0)`
- スコア計算式
  - `D = round(100 * (0.20*b + 0.35*a + 0.25*c + 0.20*u))`
- ランク閾値
  - Easy: `0 <= D <= 24`
  - Normal: `25 <= D <= 49`
  - Hard: `50 <= D <= 74`
  - Expert: `75 <= D <= 100`
- 論理解法可否が False の場合でもランクは 4段階のいずれかを返し、`requires-guess=true` を補助指標として表示する

# MVP 範囲（初版で作る範囲）

- 任意サイズ（最小 2x2、最大 50x50）の盤面生成
- 開始マスの指定機能
- 開始マス安全制約（開始マス + 周囲 8 マスへの爆弾配置禁止）
- 固定マスサイズでの描画
- 盤面オーバーフロー時の縦横スクロール対応
- 爆弾トグル編集
- 数字の常時表示（隠さない）
- 爆弾配置変更時の数字即時更新
- 難易度判定（4段階: Easy/Normal/Hard/Expert）
  - 入力: 盤面情報
  - 出力: ランク（Easy/Normal/Hard/Expert）と根拠値（`D`, `B`, `A`, `C`, `U`, `requires-guess`）
- 論理解法可否判定
  - 仮定なしで、基本ルール + 高度ルール適用時に進行不能にならず完了できるかを判定
  - 判定結果を True/False で表示

# 非MVP範囲（今回は作らない範囲）

- 画像テーマ切替、音声、演出
- オンライン共有、ランキング、認証
- 高度な作問支援（最小手数自動生成、唯一解保証、複数解の分類）
- スマホ向けジェスチャー最適化

# 技術要件（Phaser/Vite/ESLint/Vitest での実装方針）

- Phaser.js + Vite 構成を維持
- Scene責務を分離し、create/update の負担を最小化
- 盤面状態は配列で明示管理し、描画更新を差分化する
- 難易度判定と論理解法可否判定は純粋関数として実装し、UIから分離する
- 難易度判定は同一盤面入力に対して決定的（deterministic）であること
- 難易度判定は根拠値（`D`, `B`, `A`, `C`, `U`, `requires-guess`）を常に返すこと
- 開始マス未指定の間は評価関数を呼び出さない（UI 側で評価実行をガードする）
- セルサイズは定数管理し、ビューポート変更時もセル自体は縮小しない
- 盤面描画領域はスクロール可能なコンテナでラップし、入力座標と盤面座標の対応を維持する
- スクロールは盤面表示エリアに限定し、ページ全体スクロールとは独立して操作可能にする
- ESLint エラー 0 を維持
- Vitest で判定ロジックと境界値を重点テスト

評価関数インタフェース（TypeScript）:

```ts
export type DifficultyRank = 'Easy' | 'Normal' | 'Hard' | 'Expert';

export type BoardCell = {
  hasMine: boolean;
};

export type BoardState = {
  width: number;
  height: number;
  cells: BoardCell[]; // row-major, length = width * height
  startIndex: number; // 0 <= startIndex < width * height
};

export type DifficultyEvidence = {
  D: number; // 0-100
  B: number; // basic rule steps
  A: number; // advanced rule steps
  C: number; // max chain depth
  U: number; // unresolvedSafe / max(totalSafe, 1), clamped to 0.0-1.0
  requiresGuess: boolean;
};

export type DifficultyEvaluation = {
  rank: DifficultyRank;
  evidence: DifficultyEvidence;
  logicallySolvable: boolean;
};

export type EvaluateDifficultyInput = {
  board: BoardState;
};

export type EvaluateDifficultyResult =
  | { ok: true; value: DifficultyEvaluation }
  | { ok: false; code: 'INVALID_BOARD' | 'INVALID_START' | 'CONSTRAINT_VIOLATION'; message: string };

export function evaluateDifficulty(input: EvaluateDifficultyInput): EvaluateDifficultyResult;
```

インタフェース要件:

- `evaluateDifficulty` は純粋関数とし、引数オブジェクトを破壊的変更しない
- 同一 `EvaluateDifficultyInput` に対し、常に同一 `EvaluateDifficultyResult` を返す
- 失敗は例外送出ではなく `ok: false` の戻り値で返す
- 評価ルールは固定（基本ルール + 高度ルール、仮定禁止）とし、呼び出し側から切り替えない
- `logicallySolvable` が `false` の場合、`evidence.requiresGuess` は必ず `true` とする
- `evidence.requiresGuess` と `logicallySolvable` は常に補集合とする（`requiresGuess = !logicallySolvable`）

論理解法ルールセット:

- 基本ルール
  - 確定開放: 数字セルの未確定近傍数と残り爆弾数から安全マスを確定する
  - 確定爆弾: 数字セルの未確定近傍数と一致条件から爆弾マスを確定する
- 高度ルール
  - 近傍集合の包含関係による差分推論
  - 複数数字セル間の制約連立による確定推論
  - 連鎖推論（新規確定結果を逐次反映して反復適用）
- 非許可
  - 仮定置き（場合分け）や乱択探索は使用しない

適用順序・停止条件（決定性担保）:

- 1 サイクルを「基本ルールを変化がなくなるまで反復 -> 高度ルールを 1 回走査」とする
- 高度ルール走査で新規確定が 1 件でも発生したら次サイクルへ進む
- 1 サイクル内で新規確定が 0 件なら停止する
- 同時に複数候補がある場合の処理順は row-major（上から下、左から右）で固定する
- ルール適用優先順位は「確定爆弾 -> 確定開放 -> 高度ルール」の順で固定する

# テスト観点（最低限の単体テスト対象）

- 数字計算
  - 爆弾なし盤面
  - 角・辺・中央の隣接数
  - 爆弾トグル後の再計算
- サイズ変更
  - 最小値・最大値・境界外入力の丸め/拒否
- 画面表示とスクロール
  - 小画面で盤面がはみ出す場合にスクロール可能
  - スクロール後もクリック対象セルが正しく一致する
  - 画面サイズ変更後もマスサイズが変化しない
  - スクロールが盤面表示エリア内に限定される
- 開始マス制約
  - 開始マス自身へ爆弾を置けない
  - 開始マスの周囲 8 マスへ爆弾を置けない
  - 盤面端・角での近傍判定（存在セルのみ）
  - 制約領域に既存爆弾がある状態で開始マス設定を拒否する
- 難易度判定
  - 4段階（Easy/Normal/Hard/Expert）以外の値を返さない
  - スコア `D` が 0-100 の範囲に収まる
  - 閾値境界（24/25, 49/50, 74/75）でランクが正しく切り替わる
  - 同一入力で常に同一の根拠値（`D`, `B`, `A`, `C`, `U`, `requires-guess`）を返す
  - `evaluateDifficulty` が入力を破壊的変更しない
  - 不正入力時に例外ではなく `ok: false` を返す
  - 開始マス未指定時は評価を実行しない（UI 側ガード）
  - `logicallySolvable=false` の場合、`requiresGuess=true` になる
  - `requiresGuess` と `logicallySolvable` が常に補集合である
  - 同一爆弾数でも配置差でランクが変わるケース
  - 空盤面・高密度盤面の扱い
- 論理解法可否判定
  - 明らかに論理で解ける盤面
  - 仮定が必要な盤面
  - 不正盤面（矛盾）
  - 高度ルールを適用した場合のみ解ける盤面
  - ルール適用順序・停止条件・row-major タイブレークで結果が一意になる

# 完了条件（受け入れ条件）

- ユーザーが幅・高さを変更すると盤面サイズが即時反映される
- ユーザーが開始マスを指定でき、指定後は開始マスと周囲 8 マスに爆弾を置けない
- 制約領域に既存爆弾がある状態で開始マス指定した場合は、設定が拒否される（自動解除しない）
- 画面サイズが変わってもマスサイズは縮小されない
- 盤面が表示領域を超える場合、盤面表示エリア内の縦横スクロールで全セルへアクセスできる
- どの時点でも全セルの数字が表示され、爆弾トグル時に即時更新される
- 難易度ランクが常に表示され、盤面変更で更新される
- 難易度の根拠値（`D`, `B`, `A`, `C`, `U`, `requires-guess`）が表示され、盤面変更で更新される
- 論理解法可否（True/False）が常に表示され、盤面変更で更新される
- 開始マス未指定時は難易度/論理解法可否を未評価状態として表示する
- 評価関数 `evaluateDifficulty` が仕様通りの型と戻り値契約（`ok` 判定）を満たす
- `npm run lint` / `npm run test` / `npm run test:coverage` / `npm run build` が成功する
- テストカバレッジ 90% 以上を満たす
