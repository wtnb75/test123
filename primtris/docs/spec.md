# ゲーム仕様書: Primtris（仮称）

## ゲーム名（仮称可）
Primtris（プリムトリス）

## コンセプト（1-3 行）
落ちてくる数字を、数の性質（素数・倍数）で素早く正しいボックスへ振り分けるパズルゲーム。
判定に成功すると投入数字の各桁が消え、さらにボックス内に溜まった同じ桁数字を巻き添えで消去できる連鎖感を狙う。
シンプルな入力と算数的判断を組み合わせ、短時間でも集中して遊べる体験を目指す。

## ターゲットプレイヤー
- 暗算や分類パズルが好きなプレイヤー
- 短時間で遊べるカジュアルゲームを求めるプレイヤー
- 小学校高学年以上の数のルールに抵抗がない層

## コアループ（プレイ中に繰り返す行動）
1. 上から数字タイルが1つ落下する
2. プレイヤーが該当ボックス（x2, x3, x5, x7, xN, P）へ入力操作で振り分ける
3. 正誤判定を行う
4. 正解なら投入数字の桁消去 + 同桁巻き添え消去、不正解なら投入数字の各桁をボックス内に残留
5. 投入完了数をスコアとして加算する
6. 次のタイルが落下する

## 操作仕様（入力デバイスと主要操作）
- 入力デバイス: タッチ（MVP）、キーボード（サブ）
- タッチ操作（MVP）:
  - 掴む: 落下中の数値をタッチ開始でホールド
  - 左右移動: ドラッグに追従して水平方向へ移動
  - 離す: 指を離した位置のレーンから落下再開し、到達ボックスへ投入
  - ガイド: 掴み中は着地点ボックスをハイライト表示
- キーボード操作（サブ）:
  - 左右移動: 落下中タイルの移動
  - 決定: 現在位置のボックスに投入
  - 一時停止: ポーズ
- 入力の優先:
  - 同時入力時は直近の入力デバイスを優先する
  - 掴み中は重力落下を一時停止し、離した瞬間に再開する
  - ボックス領域の外側で指を離した場合は「掴み継続」のまま停止し、再タッチがあるまで落下再開しない
  - ドロップ先の判定は x 座標のみで行う

## 画面・Scene 構成
- Boot/Preload Scene: アセット読み込み
- Title Scene: タイトル表示、開始操作
- Game Scene:
  - 落下中タイル
  - 現在操作中の数値（CURRENT表示）
  - 次に落下する数値（NEXT 2件表示）
  - ボックス列（x2, x3, x5, x7, xN, P）
  - スコア、各ボックス残留桁数/容量表示
- Result Scene: スコア、リトライ導線

## 画面イメージ（ワイヤー）

### Title Scene
```text

|                                                      |
|                 [ Press Enter ]                      |

|   Rule: choose x2/x3/x5/x7/xN/P for each number     |
+------------------------------------------------------+
```

### Game Scene（メイン画面）
```text
+--------------------------------------+
| SCORE 1240                           |
| CURRENT: 84                          |
| NEXT: [ 135 ] [ 462 ]               |
|--------------------------------------|
|                8                     |
|                4                     |
|            (drag to move)            |
|                 ||                    |
|                 vv                    |
|--------------------------------------|
| [x2][x3][x5][x7][xN][P ]             |

| 9/12 8/12 0/12 2/12 3/12 3/12        |
```

### Result Scene
```text
+------------------------------------------------------+
|                    RESULT                            |
|                                                      |
|   SCORE: 004980                                      |
|   COLLATERAL CLEAR: 31                              |
|                                                      |
|   [ Retry ]                  [ Back to Title ]       |
+------------------------------------------------------+
```

補足:
- Game Scene の各ボックス内表示は「残留している桁数字のリスト」を示す
- CURRENT欄には、現在ドラッグ中または落下中の数値を横書きで表示する
- NEXT欄には、現在操作中の数値の次に投入される予定の数値を2件表示する
- 正解時は投入数字の各桁は残らず、同じボックス内の一致桁を巻き添え消去する
- 失敗時は投入数字を桁分解して、投入先ボックスに追加する
- 各ボックスには容量（残留可能な桁数上限）があり、超過した時点でゲームオーバー
- スマートフォン縦画面を基準とし、ボックスは 6列 x 1行 を優先配置する
- ボックス同士の境界線の幅は 0 とし、見た目も入力判定も隙間なく横並びにする
- 落下中の数値をボックス領域の外側で離した場合は、掴み継続のまま停止し、再タッチまで待機する
- 6列表示を成立させるため、各ボックスの常時表示は「ラベル + 圧縮残留表示 + 残留/容量」に限定する
- 詳細な残留桁リストはタップ時のポップアップまたは下段詳細エリアで表示する
- 落下中数値は桁数に関わらず常に縦積み表示とする
- 数値としての認識を補助するため、同じ値を CURRENT欄に横書きでも表示する

## ルール（勝利条件、失敗条件、スコア条件）
- 分類先ルール:
  - 2の倍数 -> x2
  - 3の倍数 -> x3
  - 5の倍数 -> x5
  - 7の倍数 -> x7
  - 上記いずれにも該当しない合成数 -> xN
  - 素数 -> P
- 出現数値ルール:
  - 出現する値は難易度で設定された桁数ちょうどの正の整数
  - 負数は出現しない
  - 0 と 1 は出現しない
  - 最も簡単な難易度は 2桁とし、1桁の 2/3/5/7 は出現しない
- 正解判定ルール（素因数一致）:
  - 素数は P のみ正解
  - 合成数は、素因数に 2/3/5/7 のいずれかを含む場合、その対応ボックスすべてが正解
  - 例: 30 は x2 / x3 / x5 のどこに入れても正解
  - 例: 42 は x2 / x3 / x7 のどこに入れても正解
  - 2/3/5/7 を素因数に持たない合成数のみ xN が正解
- 残留単位ルール:
  - ボックスに残る単位は「入力値そのもの」ではなく「各桁の数字」
  - 例: 14 を誤って x3 に入れた場合、x3 には 1 と 4 が残る
- 正解時:
  - 投入した数字の各桁は消える（残留しない）
  - 投入先ボックス内に溜まっている桁数字のうち、投入数字の各桁と同じ数字を巻き添えで消去する
  - 判定は桁の位置に依存しない（同じ数字であれば位置が違っても消去対象）
  - 巻き添え消去は 1対1 で行い、投入桁1個につき残留桁を最大1個だけ消せる
  - 巻き添えの対応順は、投入数字の上の桁から順に処理する
  - 残留側に同じ数字が複数ある場合は、新しく追加されたものから順に消去する
  - 例: x3 に 1 と 4 が残っている状態で 12 を正解投入した場合、12 の 1 と 2 は消え、残留していた 1 と 4 のうち 1 が巻き添えで消える（4 は残る）
  - 例: 残留 [1,1,4] に対して 12 を正解投入した場合、巻き添えで消える 1 は1個のみで、結果は [1,4]
- 不正解時:
  - 投入した数字の各桁が、投入先ボックスに残る
  - 残留桁数が増え、盤面圧迫リスクが上がる
- 失敗条件（MVP）:
  - 不正解による桁追加直後、いずれか1つのボックスで残留桁数がそのボックス容量を超える
- スコアルール:
  - スコアは投入完了した数値の個数のみで決まる
  - 正解/不正解、巻き添え消去数、連続成功による追加点はない
- プレイ継続:
  - 制限時間は設けない
  - ゲームオーバーになるまで継続してプレイできる
  - Result Scene はゲームオーバー時のみ表示する

## 難易度調整
- 難易度パラメータ:
  - 出現数値の桁数（digitCount）
  - 各ボックス容量（boxCapacity）
- 基本方針:
  - 桁数が大きいほど、誤投入時に一度に残る桁数が増えて難しくなる
  - ボックス容量が小さいほど、ミス許容量が減って難しくなる
- MVPの初期プリセット案:
  - Easy: 桁数 2、boxCapacity 12
  - Normal: 桁数 3、boxCapacity 9
  - Hard: 桁数 4、boxCapacity 7
- ステージ進行（任意）:
  - まずは固定難易度で開始する
  - ゲーム性に応じて将来、digitCount や boxCapacity の段階変化を再検討する

## 技術要件（Phaser/Vite/ESLint/Vitest での実装方針）
- Phaser.js:
  - Scene責務分離（初期化/進行/終了）
  - updateでの処理を最小化し、判定ロジックは純関数化
- Vite:
  - `npm run build` で静的配布可能
  - アセット参照はビルド後パス互換を維持
- ESLint:
  - lintエラー0を維持
  - 例外コメントは最小範囲かつ理由明記
- Vitest:
  - 分類判定関数、桁分解残留処理、巻き添え消去判定、スコア計算を単体テスト
  - カバレッジ90%以上を目標に回帰ケースを追加

## テスト観点（最低限の単体テスト対象）
- 分類判定:
  - 生成対象の素数（例: 11, 13, 17...）がPに入ること
  - 合成数の複数正解判定（例: 30はx2/x3/x5、42はx2/x3/x7）
  - xN判定（例: 121, 169 など 2/3/5/7 非該当の合成数）
- 数値生成:
  - 難易度で指定した桁数ちょうどの正の整数のみ出現すること
  - 0、1、負数、1桁数が出現しないこと
- 正誤処理:
  - 正解時に投入数字の各桁が残留しないこと
  - 不正解時に投入数字が桁分解されて残ること（例: 14 -> 1 と 4）
- 入力操作:
  - ドラッグ中は数値が水平方向に追従すること
  - ドロップ時に現在レーンのボックスへ正しく投入されること
  - 掴み中に落下が停止し、離した後に再開すること
  - ボックス外で離した場合は掴み継続のまま停止し、再タッチまで再開しないこと
  - ドロップ判定が x 座標のみで行われること
- 巻き添え消去:
  - 正解時に投入桁と同じ数字の残留桁が巻き添えで消えること（桁位置は不問）
  - 巻き添え消去が 1対1 であること（投入桁1個につき残留桁1個まで）
  - 巻き添え消去が投入数字の上の桁から順に処理されること
  - 同じ数字が複数残っている場合は新しいものから消えること
  - 例: x3 に [1,4] 残留時、12 を正解投入すると [4] になること
  - 例: x3 に [1,1,4] 残留時、12 を正解投入すると [1,4] になること
  - 一致桁がない場合は追加消去なし
- 終了判定:
  - 不正解で桁追加した直後に容量超過したらゲームオーバー
- 難易度:
  - 難易度別に digitCount が変わること
  - 難易度別に boxCapacity が変わること
- 表示:
  - 落下中数値が常に縦積み表示されること
  - CURRENT欄に同じ値が横書き表示されること
- NEXT表示:
  - 次に落下する数値が2件表示されること
  - 現在の数値投入後にNEXTが1件ずつ繰り上がること
- スコア:
  - スコアが投入完了数と一致すること

## 完了条件（受け入れ条件）
- 仕様どおりに6分類ボックスへ投入判定が動作する
- NEXT欄に次の数値が2件表示され、現在の数値投入後に正しく繰り上がる
- 正解時に投入桁消去と同桁巻き添え消去が動作する
- 不正解時に数字が桁分解されてボックスに残る
- 最低限のUI（スコア、各ボックス残留桁数/容量、終了状態）が表示される
- 不正解による桁追加直後の容量超過で確実にゲームオーバーになる
- スコアが投入完了数のみで計算される
- Result Scene がゲームオーバー時のみ表示される
- `npm run lint` / `npm run test` / `npm run test:coverage` / `npm run build` が通る
- テストカバレッジ90%以上を満たす（未達時は不足ケースと補完計画を明示）
