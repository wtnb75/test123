# ゲーム仕様書: 古語ドロップ（KogoDrop）

## ゲーム名（仮称可）
古語ドロップ（KogoDrop）

## コンセプト（1-3 行）
高校受験対策向けの古語単語暗記ゲーム。
上から落ちてくる古語の単語を、画面下部に並ぶ意味スロットへ正しく振り落とす落ちゲー。
言語モード（古語↔現代語 / 古語↔英語 / 英語↔古語）を切り替えることで、古語・現代語・英語を同時に学べる。

## ターゲットプレイヤー
- 高校受験を控えた中学生
- 古文の語彙力を短時間で鍛えたいプレイヤー
- 英単語と古語を並行して学びたいプレイヤー
- 落ちゲーのシンプルな操作感が好きなプレイヤー

## コアループ（プレイ中に繰り返す行動）
1. 上から単語タイルが1つ落下する（表示内容は言語モードによって異なる）
2. プレイヤーが該当するスロットへ左右移動で振り分ける
3. タイルがスロットに着地した瞬間に正誤判定する
4. 正解なら正答数 +1、不正解なら不正解数 +1（タイルと正解をフラッシュ表示）
5. 出題数カウンタが進み、次の単語タイルが落下する
6. 規定出題数に達したらリザルト画面へ

## 操作仕様（入力デバイスと主要操作）
- 入力デバイス: タッチ（MVP）、キーボード（サブ）
- タッチ操作（MVP）:
  - 掴む: 落下中のタイルをタッチ開始でホールド
  - 左右移動: ドラッグに追従して水平方向へ移動
  - 離す: 指を離した位置のスロットへ落下・着地
  - ガイド: 掴み中は着地点スロットをハイライト表示
- キーボード操作（サブ）:
  - 左右矢印キー: 落下中タイルの移動
  - スペースキー / Enter: 現在位置のスロットへ即時落下
  - P キー: 一時停止
- 入力の優先:
  - 同時入力時は直近の入力デバイスを優先する
  - 掴み中は重力落下を一時停止し、離した瞬間に再開する
  - スロット領域の外側で指を離した場合は「掴み継続」のまま停止し、再タッチまで待機

## 画面・Scene 構成
- Boot/Preload Scene: アセット読み込み
- Title Scene: タイトル表示、言語モード選択、難易度選択、出題数選択（10 / 20 / 30問）、開始操作
- Game Scene:
  - 落下中のタイル（言語モードによって表示内容が変わる）
  - 進捗表示（n問目 / 全問数）
  - 次に落下する単語プレビュー（NEXT 1件）
  - 意味スロット列（4列固定、言語モードによって表示内容が変わる）
  - 正答数 / 出題済み数
  - 正誤フラッシュ表示エリア
- Result Scene: 正答数、不正答数、正答率、復習ボタン（不正解が1問以上の場合）、リトライ導線、タイトルへ戻る導線
- Review Scene: 不正解単語の一覧表示（古語 → 正しい意味）、タイトルへ戻る導線

### Game Scene（メイン画面）

言語モードによってタイルとスロットの表示内容が変わる。

**古語→現代語モード**（デフォルト）
```text
+--------------------------------------+
| 3問目 / 20問   正答: 2 / 出題: 2     |
| NEXT: [をかし]                       |
|--------------------------------------|
|           ┌────────┐                 |
|           │ あはれ  │  ← 古語        |
|           └────────┘                 |
|                ↓↓                    |
|--------------------------------------|
| [しみじみとした] [趣がある] [非常に] [かわいい] |
+--------------------------------------+
| ✓ あはれ → しみじみとした感動         |
+--------------------------------------+
```

**古語→英語モード**
```text
+--------------------------------------+
| 3問目 / 20問   正答: 2 / 出題: 2     |
| NEXT: [をかし]                       |
|--------------------------------------|
|           ┌────────┐                 |
|           │ あはれ  │  ← 古語        |
|           └────────┘                 |
|                ↓↓                    |
|--------------------------------------|
| [  pathos  ] [charming] [  very  ] [lovely] |
+--------------------------------------+
| ✓ あはれ → pathos; touching sentiment |
+--------------------------------------+
```

**英語→古語モード**
```text
+--------------------------------------+
| 3問目 / 20問   正答: 2 / 出題: 2     |
| NEXT: [charming]                     |
|--------------------------------------|
|           ┌────────┐                 |
|           │ pathos │  ← 英語        |
|           └────────┘                 |
|                ↓↓                    |
|--------------------------------------|
| [ あはれ ] [ をかし ] [ いみじ ] [ かなし ] |
+--------------------------------------+
| ✓ pathos → あはれ                    |
+--------------------------------------+
```

補足:
- スロットは画面幅を均等に4分割して横並びに配置する
- 各スロットには対応フィールドの短縮値（`shortMeaning` / `shortEnglishMeaning` / `word`）を常時表示する
- スマートフォン縦画面を基準とする

```text
|                    リザルト                           |
|                                                      |
|   出題数:   20問                                     |
|   正答数:   15問                                     |
|   不正答数:  5問                                     |
```

補足:

- 「もう一度」ボタンは言語モード・難易度・出題数を引き継いだまま Game Scene を再開する（Title Scene には戻らない）

### Review Scene
```text
+------------------------------------------------------+
|                  復習 — 間違えた単語                  |
|                                                      |
|   1. おぼつかなし［形容詞・ク活用］                  |
|      → 不安だ・はっきりしない                         |
|      → uneasy; vague; unreliable                    |
|      例）「返り事もおぼつかなくて、日ごろ過ぎぬ。」    |
|          （返事もはっきりせず、何日も過ぎた。）        |
|          ※「おぼつかなく」= 連用形                   |
|                                                      |
|   2. いみじ［形容詞・シク活用］                      |
|      → 非常に・ひどい・すばらしい                     |
|      → very; terribly; wonderful                    |
|      例）「いみじう美しき人の…」                      |
|          （非常に美しい人が…）                        |
|          ※「いみじう」= 連用形・ウ音便               |
|                                                      |
|   3. をかし［形容詞・シク活用］                      |
|      → 趣がある・おもしろい                           |
|      → charming; interesting; witty                 |
|      例）「春はあけぼの。…紫だちたる雲の…」           |
|          （春は夜明け。…紫がかった雲が…）             |
|          ※出典: 枕草子                               |
|                                                      |
|   （スクロールで全件表示）                            |
|                                                      |
|   [ タイトルへ ]                                     |
+------------------------------------------------------+
```

補足:
- 不正解単語は出題順に並べる
- 各エントリは「古語［品詞・活用種別］」と `meaning`（フル表記）と `englishMeaning` を表示する
- その単語を `highlights.word` に含む例文を最大1件表示する（複数ある場合は先頭1件）
- 例文がある場合、該当 `highlight` の `note` を「※「form」= note」の形式で表示する
- 例文の `source` がある場合は「※出典: 〇〇」を追加表示する
- 例文が存在しない単語は意味・英語訳・品詞のみ表示する
- スクロール可能なリスト形式にする（単語数が多くなる場合を考慮）
- スクロールは DOM オーバーレイで実装する（Phaser キャンバスの上に HTML `<div>` をかぶせ、CSS `overflow-y: auto` でスクロール）
- Review Scene 開始時に DOM 要素を生成し、Scene 終了時（タイトルへ戻るボタン押下時）に削除する

## 古語単語データ仕様

### 単語構造（`KogoEntry`）
各エントリは以下を持つ:
- `id`: 一意の識別子（例: `"aware"`）
- `word`: 古語（終止形。ひらがな表記を基本とするが、漢字仮名交じり表記も可。例: `"あはれ"` または `"哀れ"`）
- `reading`: ひらがな読み（`word` がひらがなのみの場合は省略可。`word` に漢字を使う場合は必須。スロット・タイル表示・`highlights` 照合はすべて `word` を使う）
- `meaning`: 主要な現代語訳（30文字以内）
- `shortMeaning`: スロット表示用の短縮訳（10文字以内）
- `level`: 難易度レベル `"basic" | "standard" | "advanced"`
- `pos`: 品詞・活用種別。以下の定義済み値のいずれかを使う:
  - 形容詞: `"形容詞・ク活用"`, `"形容詞・シク活用"`
  - 形容動詞: `"形容動詞・ナリ活用"`, `"形容動詞・タリ活用"`
  - 動詞: `"動詞・四段活用"`, `"動詞・上一段活用"`, `"動詞・上二段活用"`, `"動詞・下一段活用"`, `"動詞・下二段活用"`, `"動詞・カ行変格活用"`, `"動詞・サ行変格活用"`, `"動詞・ナ行変格活用"`, `"動詞・ラ行変格活用"`
  - その他: `"名詞"`, `"副詞"`, `"助動詞"`, `"接続詞"`, `"感動詞"`
- `englishMeaning`: 英語訳（フル表記・50文字以内、例: `"pathos; touching sentiment"`）
- `shortEnglishMeaning`: スロット表示用の短縮英語訳（15文字以内、例: `"pathos"`）
- `verified`: レビュー済みフラグ（`boolean`）。`false` のエントリは出題プールから除外する

#### レベル定義
| level | 目安 | 語数目標 |
|-------|------|---------|
| `basic` | 高校受験の最頻出語。ほぼ全員が学ぶ | 約60語 |
| `standard` | 高校受験の標準語。公立上位〜私立を想定 | 約60語 |
| `advanced` | やや難しい語。難関私立・大学受験入口レベル | 約40語 |
| **合計** | | **約160語** |

### 例文構造（`ExampleSentence`）
単語とは独立したコレクションとして `data/exampleSentences.ts` に格納する。
各エントリは以下を持つ:
- `sentence`: 古文テキスト
- `translation`: 現代語訳
- `highlights`: この例文が主に示す単語の情報オブジェクトの配列（1件以上）。各オブジェクトは:
  - `word`: 終止形（`KogoEntry.word` と一致、逆引きに使う）
  - `form`: 例文中の実際の語形（`sentence.includes(form)` が成立する文字列）
  - `note?`: 活用説明（例: `"連用形・ウ音便"`, `"連体形"`, `"連用形"` 等）
- `source?`: 出典（古典文学からの引用の場合のみ記載。例: `"枕草子"`, `"源氏物語"`）
- `verified`: レビュー済みフラグ（`boolean`）。`false` の例文は Review Scene に表示しない

`highlights` をオブジェクト配列にすることで:
- `sentence.includes(h.form)` による確実な一致（辞書形ではなく実際の語形を使う）
- 単語からの逆引き: `exampleSentences.filter(ex => ex.highlights.some(h => h.word === entry.word))`
- `note` で活用の知識をプレイヤーに渡せる

単語と例文の関係は多対多:
- 1つの例文が複数の単語を示せる（`highlights` に複数のオブジェクト）
- 1つの単語に複数の例文を当てられる（別の例文エントリで同じ `word` を参照）

### データ検証ワークフロー

#### チェックが必要な箇所とリスク
| フィールド | リスク | 理由 |
|-----------|--------|------|
| `pos` | 高 | シク活用 vs ク活用の区別、形容動詞の種別など微妙なケースがある |
| `highlights[].note` | 高 | 文法用語の誤用・活用分類のミスが起きやすい |
| 生成 `sentence` | 中 | 助動詞の誤用・不自然な古文が混入しうる |
| `meaning` | 中 | 複数義語の主義・副義の扱いが辞書によって異なる |
| `englishMeaning` / `shortEnglishMeaning` | 中 | 古語の多義性を英語で正確に表現するのが難しく、誤訳・意味の欠落が起きやすい |
| 引用 `sentence` | 低 | 出典が明確なら本文は正確。ただし `translation` は要確認 |

#### 参照する辞書（正とする権威）
- **旺文社全訳古語辞典**（高校受験〜大学受験向け、入手しやすい）
- **学研全訳古語辞典**（受験向け、オンライン版あり）
- いずれかを「正」として決め、意味・品詞の表記を統一する

#### 自動チェック（テストで機械的に担保する）
- `sentence.includes(h.form)` が全エントリで成立すること
- `pos` が定義済み enum の値であること
- `meaning` 30文字以内・`shortMeaning` 10文字以内であること
- `englishMeaning` 50文字以内・`shortEnglishMeaning` 15文字以内であること
- `highlights[].word` が `kogoList` の `word` と一致すること
- `verified: true` のエントリのみが出題プール・Review Scene に使われること

#### 人間レビューの進め方
新規エントリは `verified: false` で追加し、辞書で確認後に `verified: true` へ変更する。
レビューは level 順（basic → standard → advanced）に進める。

```
Phase 1: basic 約60語 + 対応例文を verified にする
Phase 2: standard 約60語 + 対応例文
Phase 3: advanced 約40語 + 対応例文
```

#### 動作に必要な最低 verified 語数

| 目的 | 最低語数 | 理由 |
|------|---------|------|
| ゲームが起動できる（技術的最低限） | 4語 | スロット4列を埋めるのに必要 |
| 10問セッションが成立する（開発テスト用） | 20語 | 繰り返しが少なくテスト可能な最低限 |
| MVP リリース（やさしいモードのみ） | basic 60語すべて | Phase 1 完了が条件 |
| MVP リリース（全モード） | basic + standard 120語 | Phase 2 完了が条件 |

開発中は `verified: false` の語を出題プールに含めるデバッグフラグを Vite の環境変数で切り替えられるようにしておくと検証作業が進めやすい。変数名は `VITE_DEV_IGNORE_VERIFIED`（`import.meta.env.VITE_DEV_IGNORE_VERIFIED`）とし、`.env.development` に `VITE_DEV_IGNORE_VERIFIED=true` を記述して有効化する。

### 例文収録方針
| 対象 | 方針 |
|------|------|
| あはれ・をかし・いみじ 等の最頻出語 | 枕草子・源氏物語・徒然草等から引用、`source` を明記 |
| それ以外の語 | 受験問題スタイルに揃えた生成文、`source` は省略 |
| 生成文で精度が不安な語 | 生成後に人間がレビューして必要なら差し替え |

- 単語: 約160語（basic 60 / standard 60 / advanced 40）
- 例文: 単語数より多くなることを想定（1語あたり1〜3例文目安）

### 例文収録例

```ts
// 古典引用（source あり）
{
  sentence: "春はあけぼの。やうやう白くなりゆく山ぎは、すこしあかりて、紫だちたる雲のほそくたなびきたる。",
```

## スロット選択肢の生成ルール

言語モードによってタイルとスロットに使うフィールドが変わる:

| モード | タイル（落下） | スロット（選択肢） | 正解判定フィールド |
|--------|-------------|-----------------|-----------------|
| 古語→現代語 | `word` | `shortMeaning` | `shortMeaning` |
| 古語→英語 | `word` | `shortEnglishMeaning` | `shortEnglishMeaning` |
| 英語→古語 | `shortEnglishMeaning` | `word` | `word` |

共通ルール:
1. 正解スロットの値を必ず1つ含める
2. その他3スロットには出題プール内の別単語の対応フィールドからランダムに選ぶ
3. 同じ値が重複しないようにする（`かなし` と `うつくし` のように `shortMeaning` が同値の語が出題セットに混在する場合は、値が重複しない候補を優先して選ぶ）
4. 正解スロットの位置は毎回ランダムに決定する
5. プール内で重複排除後の候補が3件未満になる場合は、重複を許容して4スロットを埋める（フォールバック）

### NEXT プレビューの表示ルール

NEXT プレビューはタイルと同じフィールドを表示する:

| モード | NEXT に表示するフィールド |
|--------|------------------------|
| 古語→現代語 | `word`（古語） |
| 古語→英語 | `word`（古語） |
| 英語→古語 | `shortEnglishMeaning`（英語） |

### 正誤フラッシュ表示のルール

フラッシュはタイル表示値と正解のフル表記を「→」でつなぐ形式で表示する:

- 正解時: `✓ [タイル表示値] → [正解フル表記]`
- 不正解時: `✗ [タイル表示値] → [選択値]（正解: [正解フル表記]）`

モード別の正解フル表記フィールド:

| モード | タイル表示値 | 正解フル表記フィールド |
|--------|------------|-------------------|
| 古語→現代語 | `word` | `meaning` |
| 古語→英語 | `word` | `englishMeaning` |
| 英語→古語 | `shortEnglishMeaning` | `word` |

例:
- 古語→現代語で正解: `✓ あはれ → しみじみとした感動・趣`
- 古語→英語で不正解: `✗ あはれ → lovely（正解: pathos; touching sentiment）`
- 英語→古語で正解: `✓ pathos → あはれ`

フラッシュ表示の時間:
- フラッシュは **2.5秒** 表示する
- 2.5秒経過後、フラッシュを消去して次のタイルの落下を開始する
- フラッシュ表示中は入力を受け付けない（次タイルはまだ存在しない）

## ルール（勝利条件、失敗条件、スコア条件）
- 出題ルール:
  - 選択した出題数（10 / 20 / 30）だけ出題する
  - 同じ単語が連続して出題されないようにする（直前1語は除く）
  - 出題済みセットからランダム非復元抽出し、全単語を一周したらシャッフルして再抽出
  - 折り返し時（シャッフル直後）に新ラウンド先頭が前ラウンド最後と同じ単語になった場合は、先頭とそれ以外のランダムな位置をスワップして連続を回避する（プールが1語のみの場合は連続を許容する）
  - 難易度ごとの出題単語プール:
    | 難易度 | 出題プール |
    |--------|-----------|
    | やさしい | `basic` のみ（約60語） |
    | ふつう | `basic` + `standard`（約120語） |
    | むずかしい | `basic` + `standard` + `advanced`（約160語） |
  - ダミー選択肢は出題プール内の単語から選ぶ（難易度が上がると紛らわしい語が混ざりやすくなる）
- 正誤判定:
  - タイルが着地したスロットの正解判定フィールドの値が、出題単語の同フィールド値と一致すれば正解
  - 判定フィールドは言語モードによって変わる（スロット生成ルールの表を参照）
- ゲーム終了条件:
  - 規定出題数に到達したらゲーム終了
  - ゲームオーバーはない（時間制限なし、ミスでもゲーム継続）
- スコア:
  - 正答率 = 正答数 ÷ 出題数 × 100（小数点第1位まで）
  - 結果画面に正答数・不正答数・正答率を表示

## 落下速度仕様
- 基本落下速度: ゆっくり（受験勉強用途なので考える時間を確保）
- タイル到達までの時間: 約4〜6秒（画面高を考慮して調整）
- 掴み中は落下停止（時間制限なし）

## MVP 範囲（初版で作る範囲）
- Title Scene（言語モード・難易度・出題数の選択付き）
- Game Scene（落下タイル、スロット4列、NEXT表示1件、進捗・正答数表示）
- 古語単語データ 約160語（`englishMeaning` / `shortEnglishMeaning` 含む）
- スロット選択肢の動的生成（正解1 + ダミー3）
- 正誤判定とフラッシュ表示
- 不正解単語リストの収集（Game Scene → Result/Review Scene へ引き渡し）
- Result Scene（正答数・不正答数・正答率・復習ボタン）
- Review Scene（不正解単語の一覧表示）
- タッチ操作（ドラッグ＆ドロップ）
- キーボード操作（矢印キー、スペース/Enter）

## 非MVP範囲（今回は作らない範囲）
- 復習モードで間違えた単語のみ再出題（落ちゲー形式での再チャレンジ）
- 出題カテゴリ選択（感情語・状態語・動詞など）
- サウンド/BGM
- オンラインランキング
- タイムアタックモード
- 複数の意味を持つ語の詳細解説ポップアップ
- 難易度別（高校入試レベル / 大学入試レベル）の単語セット

## 技術要件（Phaser/Vite/ESLint/Vitest での実装方針）
- Phaser.js:
  - Scene 責務分離（Boot/Title/Game/Result/Review）
  - `preload`, `create`, `update` の責務を混在させない
  - ゲームロジック（単語抽選、正誤判定、スロット選択肢生成）は純関数として `logic/` に切り出す
  - 入力イベントは Scene 終了時に後始末する
  - Game Scene は不正解単語リスト（`KogoEntry[]`）を保持し、Result/Review Scene へ Phaser の Scene data（`this.scene.start('Result', { ... })`）で引き渡す
- Vite:
  - `npm run build` で静的配布可能
  - 単語データは TypeScript の配列定数として `data/kogoList.ts` に格納する
  - 例文データは `data/exampleSentences.ts` に格納し、`highlights[].word` で `KogoEntry.word`（日本語語形）を参照する
  - 例文の引き当てロジック（`entry.word` で `highlights` を検索）は `logic/` の純関数として実装する
- ESLint:
  - lint エラー 0 件を維持
- Vitest:
  - ロジック層（単語抽選、スロット生成、正誤判定）を単体テスト
  - カバレッジ 90% 以上

## テスト観点（最低限の単体テスト対象）
- 単語抽選:
  - 出題数ぶんの単語が重複なく（ほぼ）抽選されること
  - 直前と同じ単語が連続しないこと
  - 全単語を周回後にシャッフルして再抽出されること
  - 難易度「やさしい」では `basic` 以外の単語が出題されないこと
  - 難易度「ふつう」では `basic` + `standard` のみが出題されること
  - 難易度「むずかしい」では全レベルが出題されること
  - `verified: false` の単語が出題プールに含まれないこと
- スロット生成:
  - 各モードで正解値（`shortMeaning` / `shortEnglishMeaning` / `word`）が必ず1スロットに含まれること
  - 4スロットの値が重複しないこと
  - 正解スロットの位置がランダムに変わること（複数回生成で偏りなし）
- 正誤判定:
  - 古語→現代語モードで正解スロットに落としたとき正解になること
  - 古語→英語モードで正解スロットに落としたとき正解になること
  - 英語→古語モードで正解スロットに落としたとき正解になること
  - どのモードでも不正解スロットに落としたとき不正解になること
- 言語モード:
  - 古語→現代語モードではタイルに `word`、スロットに `shortMeaning` が使われること
  - 古語→英語モードではタイルに `word`、スロットに `shortEnglishMeaning` が使われること
  - 英語→古語モードではタイルに `shortEnglishMeaning`、スロットに `word` が使われること
- スコア:
  - 正答率が `正答数 / 出題数 × 100` で計算されること
  - 全問正解で 100.0%、全問不正解で 0.0% になること
- 出題カウンタ:
  - 出題ごとにカウンタが +1 されること
  - 規定出題数に達したらゲーム終了フラグが立つこと
- 不正解リスト収集:
  - 不正解時に当該単語が不正解リストへ追加されること
  - 同じ単語を複数回間違えた場合は重複して追加されること（出題順を保持するため）
  - 全問正解の場合は不正解リストが空であること
- 例文引き当て:
  - 単語の `word` を `highlights[].word` に含む例文が返ること
  - `verified: false` の例文が引き当て結果に含まれないこと
  - 該当例文が複数ある場合は先頭1件を返すこと
  - 該当例文がない場合は `undefined` を返すこと
  - `source` フィールドが存在する例文と存在しない例文の両方を正しく返すこと
  - `highlights[].form` が `sentence` に完全部分一致すること（活用形の検証）
  - `note` がある場合は Review Scene に表示され、ない場合は表示されないこと
- データ整合性（全データを対象にしたスタティックテスト）:
  - すべての `highlights[].word` が `kogoList` の `word` と一致すること
  - すべての `highlights[].form` が対応する `sentence` に含まれること
  - `pos` が定義済み enum 値であること
  - `meaning` が30文字以内、`shortMeaning` が10文字以内であること
  - `englishMeaning` が50文字以内、`shortEnglishMeaning` が15文字以内であること

- タイトル画面で言語モード（古語→現代語 / 古語→英語 / 英語→古語）・難易度・出題数を選べる

- スロット4列に正解1件＋ダミー3件が毎問動的に生成される

- `npm run lint` / `npm run test` / `npm run test:coverage` / `npm run build` がすべて通る
