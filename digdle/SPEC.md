# digdle 仕様案 v0.3

## 1. コンセプト

- Wordle 風の数当てゲーム。
- お題は N 桁の素数。
- プレイヤーも N 桁の素数を入力して推理する。

## 2. ゲームルール

- N は `2, 3, 4, 5` から選択する。
- 正解は「N 桁かつ素数」のランダムな 1 つ。
- 1 回の入力も「N 桁かつ素数」でなければならない。
- 回答回数は以下。
  - N=2: 5 回
  - N=3: 6 回
  - N=4: 7 回
  - N=5: 8 回
- 制限回数以内に正解すれば勝利。超えたら敗北。

## 3. 判定仕様（色ルール）

各桁を次の 3 状態で表示する。

- 緑: 数字も位置も一致
- 黄: 数字は含まれるが位置が不一致
- 灰: その数字は使われていない（または使用上限を超過）

### 3.1 重複数字の厳密ルール

同一数字が複数回出るケースで Wordle と同等に判定する。

1. まず位置一致（緑）を先に確定し、正解側の使用可能数を減らす。
2. 次に未確定桁を左から走査し、正解側に残数がある数字のみ黄にする。
3. 残数がない同数字は灰にする。

例:

- 正解 `1013`、入力 `1111`
- 判定: `緑, 灰, 緑, 灰`

## 4. 入力バリデーション

以下は不正入力として弾く（試行回数は消費しない）。

- 桁数が N と一致しない
- 素数でない

### 4.1 入力段階での制御

- 数字以外は入力段階で受け付けない（盤面に入れない）。
- 先頭桁に `0` は入力段階で受け付けない（盤面に入れない）。

### 4.2 Enter 時の検証順序

1. 桁数が N と一致するかを検証する。
2. 素数かどうかを検証する。

UI 上は理由を短く表示する。

- `N桁の数字を入力してください`
- `素数を入力してください`

## 5. 画面仕様（最小）

- タイトル: `digdle`
- N 選択 UI（2/3/4/5）
- 盤面: `試行回数 x N 桁` のマス
- 数字入力（キーボード）
  - `0-9`: 入力
  - `Backspace`: 1 文字削除
  - `Enter`: 判定実行
- 仮想キーボード（画面ボタン）
  - `0-9`: 入力
  - `削除`: 1 文字削除
  - `確定`: 判定実行
- 進行表示
  - 現在試行 `3 / 7` など
  - 勝利/敗北メッセージ
  - 正解表示（敗北時）
- リプレイ
  - 終了状態では入力をロックする
  - 終了後に任意のキーまたはボタン入力で次ゲームを開始する
  - 終了後の最初の入力は開始トリガーとしてのみ扱い、次ゲームの1文字目にはしない
  - 次ゲームの正解は直前ゲームと異なる素数を選ぶ
  - 同じ N で続行、または N を再選択できる

## 6. データ・ロジック仕様

- 起動時に N ごとの素数リストをメモリに準備する。
- 正解抽選は均等ランダム。
- テスト時は乱数シードを固定できるようにする（再現可能な結果を保証）。
  - 乱数生成器は `createGame({ seed?: number })` の形で注入可能とする。
  - `seed` 未指定時は通常乱数を使用する。
- 判定関数は副作用なしの純関数として分離する。
  - 入力: `answer: string`, `guess: string`
  - 出力: `('green' | 'yellow' | 'gray')[]`

## 7. 受け入れ条件

- 任意の N で、正解が必ず N 桁素数である。
- 不正入力が試行回数を消費しない。
- Enter 時の検証順序が「桁数 -> 素数判定」である。
- 数字以外と先頭 `0` が入力段階で拒否される。
- 重複数字の判定が 3.1 に一致する。
- 勝敗が回数条件どおりに確定する。
- 終了状態で入力がロックされ、キーまたはボタン操作で次ゲームへ遷移する。
- 終了後の最初の入力は開始トリガー専用で、次ゲームの入力文字としては消費しない。
- 次ゲームの正解が直前と同一にならない。
- テストで固定シード時に正解選択が再現可能である。
- `seed` の有無で乱数挙動が切り替わる（指定時は決定的、未指定時は非決定的）。
