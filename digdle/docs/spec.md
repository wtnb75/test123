# ゲーム名（仮称可）

digdle

# コンセプト（1-3 行）

Wordle 風の数当てゲーム。
お題は N 桁（2/3/4/5）の素数で、プレイヤーも N 桁の素数を入力して推理する。
重複数字判定は Wordle と同等ルールに従う。

# ターゲットプレイヤー

- 数字パズルが好きなプレイヤー
- 短時間で遊べる推理ゲームを求めるプレイヤー
- Wordle 系の手触りで数当てを楽しみたいプレイヤー

# コアループ（プレイ中に繰り返す行動）

1. N 桁（2/3/4/5）を選択する
2. N 桁の素数を入力する
3. Enter で判定し、色ヒント（green/yellow/gray）を読む
4. ヒントを使って次の素数を入力する
5. 制限回数内の正解を目指す

# 操作仕様（入力デバイスと主要操作）

- 入力デバイス: キーボード、画面上の仮想キー
- 数字キー（0-9）: 1 文字入力
- Backspace / ⌫: 1 文字削除
- Enter / ⏎: 判定実行
- 入力制約: 先頭 0 と非数字文字は入力不可
- ナビゲーション:
  - 左上 `🏠`: 1 階層上へ遷移
  - 右上 `🍥`: 桁数選択へ戻る

# 画面・Scene 構成

- GameScene（単一 Scene）
  - 桁数選択 UI
  - 盤面（試行回数 x N 桁）
  - 仮想キーボード
  - 結果表示（Clear / GameOver）
- レイアウト:
  - 縦長: 下部ドック型キーボード
  - 横長: 右側ドック型キーボード
  - リサイズ/回転時に再レイアウト

# ルール（勝利条件、失敗条件、スコア条件）

- 勝利条件: 制限回数内に正解の N 桁素数を当てる
- 失敗条件: 制限回数を使い切る
- 試行回数:
  - N=2: 5 回
  - N=3: 6 回
  - N=4: 7 回
  - N=5: 8 回
- 不正入力（桁不一致/非素数）は試行回数を消費しない
- 終了後は入力ロックし、次入力で同じ N の次ゲームへ進む

# MVP 範囲（初版で作る範囲）

- N 選択（2/3/4/5）
- 盤面表示と入力（物理キー/仮想キー）
- 素数検証と Enter 時検証順序（桁数 -> 素数）
- Wordle 互換の重複数字判定
- 勝敗判定（Clear / GameOver）
- 同一 N でのリプレイ、直前と異なる正解選択
- レスポンシブレイアウト（縦/横回転追従）

# 非MVP範囲（今回は作らない範囲）

- 日替わり問題
- ハードモード
- 戦績保存（勝率/連勝）
- SNS 共有
- サウンド/演出強化

# 技術要件（Phaser/Vite/ESLint/Vitest での実装方針）

- Phaser + Vite 構成を維持
- ゲームロジックは純関数として分離
  - 素数判定/生成
  - 判定ロジック
  - 入力検証
  - 状態遷移
- 乱数は `createGame({ seed?: number })` で注入可能
- 画面リサイズ時は Scene を再レイアウトし、干渉を回避
- ESLint エラー 0 を維持
- Vitest でロジック中心にテスト

# テスト観点（最低限の単体テスト対象）

- 重複数字判定（例: 1013 vs 1111）
- Enter 時の検証順序（桁数先行）
- 不正入力で試行回数を消費しない
- seed 指定時の決定性 / 未指定時の非決定性
- 次ゲームで直前解答を避ける
- レイアウト変更でゲームロジックが壊れない

# 完了条件（受け入れ条件）

- N 桁選択とゲーム開始が正常に動作する
- 判定・入力制約・勝敗が仕様どおり動作する
- 終了後遷移（同一 N 続行 / 桁数再選択）が動作する
- 画面回転・リサイズでレイアウトが破綻しない
- `npm run lint` / `npm run test` / `npm run test:coverage` / `npm run build` が成功する
- カバレッジ 90% 以上を満たす
